## Overview

Render provides a simple way to deploy both the backend and frontend services. This guide walks through deploying:

- **Backend:** Node.js API server
- **Frontend:** React/Vite static site
- **Database:** PostgreSQL (via Render)

<Warning>
  Render's free tier services will "sleep" after 15 minutes of inactivity. The first request after sleep will take longer. Consider upgrading to a paid plan for production.
</Warning>

## Prerequisites

- A [Render account](https://render.com)
- Your code pushed to a Git repository (GitHub, GitLab, or Bitbucket)
- Access to environment variable values (API keys, secrets, etc.)

## Database Setup

<Steps>
  <Step title="Create PostgreSQL database">
    1. Go to the [Render Dashboard](https://dashboard.render.com)
    2. Click **New +** → **PostgreSQL**
    3. Configure the database:
       - **Name:** `blackmichi-db`
       - **Database:** `blackmichiestudio`
       - **User:** `postgres`
       - **Region:** Choose closest to your users
       - **Plan:** Free or paid tier
    4. Click **Create Database**
  </Step>

  <Step title="Note the connection details">
    After creation, Render provides:
    - **Internal Database URL:** Use this for backend connection
    - **External Database URL:** Use for external tools

    Copy the **Internal Database URL** - you'll need it for the backend configuration.

    Example format:
    ```
    postgresql://postgres:password@dpg-xxxxx-a.oregon-postgres.render.com/blackmichiestudio
    ```
  </Step>
</Steps>

## Backend Deployment

<Steps>
  <Step title="Create web service">
    1. In Render Dashboard, click **New +** → **Web Service**
    2. Connect your Git repository
    3. Configure the service:
       - **Name:** `blackmichi-backend`
       - **Region:** Same as database
       - **Branch:** `main` (or your production branch)
       - **Root Directory:** `backend` (if backend is in subdirectory)
       - **Runtime:** Node
       - **Build Command:** `npm install`
       - **Start Command:** `node index.js`
  </Step>

  <Step title="Configure environment variables">
    In the **Environment** section, add the following variables:

    ```bash
    # Database Connection (use Internal Database URL from Render)
    DATABASE_URL=postgresql://postgres:password@dpg-xxxxx-a.oregon-postgres.render.com/blackmichiestudio

    # Server Configuration
    NODE_ENV=production
    PORT=10000

    # Frontend URL (will update after creating frontend)
    FRONTEND_URL=https://your-frontend.onrender.com

    # Authentication
    JWT_SECRET=your_secure_random_jwt_secret_here

    # Google OAuth (optional)
    GOOGLE_CLIENT_ID=your_google_client_id
    GOOGLE_CLIENT_SECRET=your_google_client_secret

    # Supabase Storage (optional)
    SUPABASE_URL=your_supabase_url
    SUPABASE_KEY=your_supabase_key

    # Payment Gateway (optional)
    FLOW_API_KEY=your_flow_api_key
    ```

    <Warning>
      **Critical Variables:**
      - `DATABASE_URL` - Must match your Render PostgreSQL Internal URL
      - `JWT_SECRET` - Use a strong random string (min 32 characters)
      - `FRONTEND_URL` - Update this after deploying frontend to fix CORS
    </Warning>
  </Step>

  <Step title="Configure health check">
    Scroll to **Health Check Path** and set:

    ```
    /health
    ```

    This allows Render to monitor if your backend is running properly.
  </Step>

  <Step title="Deploy the backend">
    Click **Create Web Service**. Render will:
    
    1. Clone your repository
    2. Install dependencies with `npm install`
    3. Start the server with `node index.js`
    4. Expose it at a Render URL like:
       ```
       https://blackmichi-backend.onrender.com
       ```

    Monitor the deployment logs. You should see:

    ```
    ╔════════════════════════════════════════════════════════════╗
    ║                    🚀 BACKEND INICIADO 🚀                  ║
    ║                   VERSIÓN: CORS ULTRA 2.0                 ║
    ║              Todos los orígenes permitidos ✅              ║
    ╚════════════════════════════════════════════════════════════╝

    ✅ Pool de conexión a BD inicializado
    ✅ Conectado a PostgreSQL
    ```
  </Step>

  <Step title="Test the backend">
    Verify the deployment:

    ```bash
    curl https://blackmichi-backend.onrender.com/health
    ```

    Expected response:
    ```json
    {
      "status": "healthy",
      "timestamp": "2026-03-13T10:00:00.000Z",
      "uptime": 12345
    }
    ```
  </Step>
</Steps>

## Frontend Deployment

<Steps>
  <Step title="Create static site">
    1. In Render Dashboard, click **New +** → **Static Site**
    2. Connect your Git repository
    3. Configure the site:
       - **Name:** `blackmichi-frontend`
       - **Branch:** `main`
       - **Root Directory:** `frontend` (if frontend is in subdirectory)
       - **Build Command:** `npm install && npm run build`
       - **Publish Directory:** `dist`
  </Step>

  <Step title="Configure environment variables">
    Add the following variable:

    ```bash
    VITE_API_BASE_URL=https://blackmichi-backend.onrender.com
    ```

    Replace with your actual backend URL from the previous section.

    <Warning>
      The `VITE_` prefix is required for Vite to expose this variable to the browser. Without it, the frontend won't connect to the API.
    </Warning>
  </Step>

  <Step title="Deploy the frontend">
    Click **Create Static Site**. Render will:
    
    1. Clone your repository
    2. Run `npm install && npm run build`
    3. Deploy the `dist` folder
    4. Expose it at a URL like:
       ```
       https://blackmichi-frontend.onrender.com
       ```
  </Step>

  <Step title="Update backend FRONTEND_URL">
    Now that you have the frontend URL, update the backend:

    1. Go to your backend service in Render Dashboard
    2. Navigate to **Environment**
    3. Update `FRONTEND_URL` to your frontend URL:
       ```
       FRONTEND_URL=https://blackmichi-frontend.onrender.com
       ```
    4. Click **Save Changes**
    5. The backend will automatically redeploy

    <Warning>
      This step is **critical** for CORS to work properly. Without it, the frontend won't be able to make API requests.
    </Warning>
  </Step>
</Steps>


Make a test request from the frontend to verify CORS and connectivity work.

## Troubleshooting

### CORS Errors

**Error:** "Not allowed by CORS" in browser console

**Solutions:**

<Steps>
  <Step title="Verify FRONTEND_URL">
    Check backend environment variables in Render Dashboard:
    - `FRONTEND_URL` must exactly match your frontend URL
    - Include `https://` prefix
    - No trailing slash
  </Step>

  <Step title="Verify VITE_API_BASE_URL">
    Check frontend environment variables:
    - `VITE_API_BASE_URL` must exactly match your backend URL
    - Include `https://` prefix
    - No trailing slash
  </Step>

  <Step title="Restart both services">
    In Render Dashboard:
    1. Go to backend service → **Manual Deploy** → **Deploy latest commit**
    2. Go to frontend service → **Manual Deploy** → **Clear build cache & deploy**
  </Step>
</Steps>

### Backend Service Slow or Timing Out

**Symptom:** First request takes 30+ seconds

**Cause:** Render free tier services sleep after 15 minutes of inactivity.

**Solutions:**
- Upgrade to a paid plan for always-on services
- Implement a keep-alive ping service to prevent sleep
- Warn users about initial load time

### Database Connection Failed

**Error:** `❌ Error inesperado en PostgreSQL` in backend logs

**Solutions:**

1. **Verify DATABASE_URL format:**
   ```
   postgresql://user:password@host:port/database
   ```

2. **Use Internal Database URL (not External):**
   - Internal URL: `dpg-xxxxx-a.oregon-postgres.render.com`
   - External URL: `dpg-xxxxx-a:5432` (don't use this for backend)

3. **Check database is running:**
   - Go to database service in Render Dashboard
   - Verify status is "Available"

4. **Check database logs:**
   - Navigate to database service → **Logs**
   - Look for connection errors

### Build Failures

**Backend build fails on `sharp` installation:**

**Solution:** Render's Node.js environment includes build tools. Check:
- Your `package.json` has correct `sharp` version (^0.34.5)
- Build command is `npm install` (not `npm ci` which may fail)
- No conflicting `sharp` installations

**Frontend build fails:**

**Solution:** Check:
- `VITE_API_BASE_URL` is set in environment variables
- Build command is correct: `npm install && npm run build`
- `dist` directory is the publish directory

### Environment Variables Not Working

**Symptom:** Backend can't read environment variables

**Solutions:**
- Verify variables are set in Render Dashboard (not in `.env` file)
- Variable names must match exactly (case-sensitive)
- Redeploy after adding/changing variables
- Check logs for "Variables de entorno faltantes" warnings

### Health Check Failing

**Error:** Render shows "Health check failed"

**Solutions:**
- Verify `/health` endpoint exists and returns 200
- Check backend logs for errors
- Test health endpoint manually: `curl https://your-backend.onrender.com/health`
- Ensure `PORT` environment variable is set (Render uses dynamic ports)

## Monitoring and Logs

### View Logs

In Render Dashboard:

1. Navigate to your service (backend or frontend)
2. Click the **Logs** tab
3. Use the search/filter to find specific errors

### View Metrics

Render provides basic metrics:

- **CPU Usage**
- **Memory Usage**
- **Request Count**
- **Response Time**

Access these in the **Metrics** tab of each service.

### Set Up Alerts

For paid plans, configure alerts:

1. Go to service → **Settings** → **Alerts**
2. Set thresholds for:
   - Health check failures
   - High CPU/memory usage
   - Deploy failures

## Production URLs

Once deployed, your production URLs will be:

- **Backend:** `https://blackmichi-backend.onrender.com`
- **Frontend:** `https://blackmichi-frontend.onrender.com`
- **API Base:** `https://blackmichi-backend.onrender.com/api`

<Warning>
  **Custom Domains:** Render allows custom domains on paid plans. Configure DNS records to point to your Render services for branded URLs like `api.blackmichi.com` and `app.blackmichi.com`.
</Warning>
