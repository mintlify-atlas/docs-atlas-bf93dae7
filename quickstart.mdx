## Quick Setup with Docker

The fastest way to get Black Michi Studio running is with Docker Compose:

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/FranciscoDastres/blackMichiEstudioPreProd.git
    cd blackMichiEstudioPreProd
    ```
  </Step>

  <Step title="Start all services">
    ```bash
    docker-compose up -d
    ```
    
    This will start:
    - PostgreSQL database
    - Backend API (Node.js/Express)
    - Frontend (React/Vite)
    - Nginx reverse proxy
  </Step>

  <Step title="Access the platform">
    Open your browser and navigate to:
    - **Frontend**: http://localhost
    - **Backend API**: http://localhost/api
    - **Health Check**: http://localhost/api/health
  </Step>
</Steps>

<Note>
  Docker Compose automatically sets up the database schema and all required services with proper networking.
</Note>

## Initial Configuration

### Create Admin Account

After starting the platform, you'll need to create an admin account:

<Steps>
  <Step title="Register a user account">
    Navigate to the registration page and create a new account.
  </Step>

  <Step title="Update user role to admin">
    Connect to the database and update the user role:
    
    ```sql
    UPDATE usuarios SET rol = 'admin' WHERE email = 'your-email@example.com';
    ```
  </Step>

  <Step title="Log in with admin privileges">
    Log out and log back in to access the admin dashboard.
  </Step>
</Steps>

### Configure Environment Variables

For production deployments, you'll need to configure environment variables:

**Backend** (`.env` in `backend/` directory):
```bash
# Database
DATABASE_URL=postgresql://user:password@host:5432/database

# JWT
JWT_SECRET=your-secret-key-here

# Flow Payment Gateway
FLOW_API_KEY=your-flow-api-key
FLOW_SECRET_KEY=your-flow-secret-key

# Google OAuth (optional)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

# Supabase Storage
SUPABASE_URL=your-supabase-url
SUPABASE_KEY=your-supabase-key
```

**Frontend** (`.env` in `frontend/` directory):
```bash
VITE_API_BASE_URL=http://localhost:3000
```

<Warning>
  Never commit `.env` files to version control. Use `.env.example` as a template.
</Warning>

## Verify Installation

Check that everything is working:

<Steps>
  <Step title="Check backend health">
    ```bash
    curl http://localhost:3000/health
    ```
    
    You should see:
    ```json
    {
      "status": "healthy",
      "timestamp": "2026-03-13T...",
      "uptime": 123
    }
    ```
  </Step>

  <Step title="Test product API">
    ```bash
    curl http://localhost:3000/api/productos
    ```
    
    Returns a list of products (may be empty initially).
  </Step>

  <Step title="Access frontend">
    Open http://localhost in your browser and verify the homepage loads.
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Port 80 already in use">
    If port 80 is already in use, edit `docker-compose.yml` and change the nginx port mapping:
    
    ```yaml
    ports:
      - "8080:80"  # Change from 80:80
    ```
    
    Then access the site at http://localhost:8080
  </Accordion>

  <Accordion title="Database connection failed">
    Make sure PostgreSQL is running and the credentials in your `.env` match your database configuration.
  </Accordion>

  <Accordion title="CORS errors in browser console">
    Ensure `VITE_API_BASE_URL` in the frontend `.env` matches your backend URL, and that the backend CORS configuration allows your frontend origin.
  </Accordion>
</AccordionGroup>
