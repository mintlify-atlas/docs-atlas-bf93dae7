# Order Tracking & History

Monitor your order status from payment confirmation through delivery with Black Michi Studio's order tracking system.

## Understanding Order States

Every order (pedido) in the system progresses through various states from creation to delivery.

### Order Lifecycle States

Based on the database schema, orders can have the following states:

| State | Spanish | Description |
|-------|---------|-------------|
| **pending** | pendiente | Order created, awaiting payment |
| **paid** | pagado | Payment confirmed by Flow |
| **confirmed** | confirmado | Order confirmed by admin |
| **processing** | en_proceso | Order is being prepared/printed |
| **shipped** | enviado | Order shipped to customer |
| **delivered** | entregado | Order delivered successfully |
| **cancelled** | cancelado | Order cancelled |
| **rejected** | rechazado | Payment rejected by Flow |

### Order State Flow

Typical order progression:

```
pendiente → pagado → confirmado → en_proceso → enviado → entregado
```

**Alternative paths:**
- **Payment Rejected**: `pendiente → rechazado`
- **User Cancelled**: `pendiente → cancelado`
- **Admin Cancelled**: `confirmado → cancelado`

<Note>
  Only orders in **"pendiente"** (pending) status can be cancelled by customers.
</Note>

## Order Creation

When you complete checkout, the system creates a new order (pedido).

### Order Information Stored

**Customer Information:**
- User ID (if logged in, NULL for guest checkout)
- Buyer name (comprador_nombre)
- Buyer email (comprador_email)
- Buyer phone (comprador_telefono)
- Shipping address (direccion_envio)

**Order Details:**
- Order ID (auto-generated)
- Commerce Order ID (unique: `BMS-{timestamp}-{random}`)
- Total amount (total)
- Shipping cost (costo_envio): CLP 3,500
- Additional notes (notas)
- Order items with quantities

**Payment Information:**
- Payment method (metodo_pago): "flow"
- Flow token (flow_token)
- Flow order number (flow_order)
- Flow status code (flow_status): 1, 2, 3, or 4
- Complete payment data (flow_payment_data): JSON

**Timestamps:**
- Created at (created_at): Order creation time
- Updated at (updated_at): Last modification time
- Payment date (fecha_pago): When payment confirmed
- Shipping date (fecha_envio): When order shipped
- Delivery date (fecha_entrega): When order delivered

### Order Items

Each product in your order is stored as a separate item (pedido_items):

- Product ID (references original product)
- Product title (at time of purchase)
- Product image (snapshot)
- Quantity ordered
- Unit price (at time of purchase)
- Subtotal (price × quantity)

<Tip>
  Product details are captured at purchase time, so you always have a record even if the product changes or is deleted later.
</Tip>

## Payment Confirmation Process

Understand how payment status is confirmed and updated.

### Webhook Confirmation

When Flow processes payment, it sends a webhook to:

**Endpoint:** `POST /api/payments/flow/confirmation`

<Steps>
  <Step title="Flow Sends Webhook">
    Flow POSTs payment confirmation to the webhook URL with token
  </Step>
  
  <Step title="Webhook Logged">
    System records webhook in `flow_webhooks` table:
    - Token
    - Request body (JSON)
    - Request headers
    - Origin IP address
    - Timestamp
  </Step>
  
  <Step title="Signature Validation">
    In production, webhook signature is validated to ensure authenticity
  </Step>
  
  <Step title="Payment Status Retrieval">
    System queries Flow API with token to get full payment status
  </Step>
  
  <Step title="Order Update">
    Order (pedido) is updated with:
    - Flow order number
    - Flow status code (1-4)
    - Mapped internal status
    - Complete payment data (JSON)
    - Payment date (if successful)
  </Step>
  
  <Step title="Stock Deduction">
    If payment successful (status = 2):
    - Stock deducted from each product
    - Product `updated_at` timestamps set
    - Warnings logged if insufficient stock
  </Step>
  
  <Step title="Webhook Marked Processed">
    Webhook record updated:
    - `processed` = true
    - `pedido_id` linked
    - `flow_order` and `flow_status` recorded
  </Step>
</Steps>

### Status Mapping

Flow status codes are mapped to internal states:

```javascript
const statusMap = {
  1: 'pendiente',  // Pending
  2: 'pagado',     // Paid
  3: 'rechazado',  // Rejected
  4: 'cancelado'   // Cancelled
};
```

<Warning>
  **Important:** Stock is only deducted after confirmed payment (status = 2), not when items are added to cart. This prevents inventory lock-up from abandoned carts.
</Warning>

## Order Status Updates

After payment confirmation, admins can update order status as it progresses.

### Status Update by Admin

Administrators can change order status through the admin panel:

**Typical Progression:**

<Steps>
  <Step title="Payment Confirmed (pagado)">
    Automatically set by webhook after successful Flow payment
    
    **Email sent:** Order confirmation
  </Step>
  
  <Step title="Order Confirmed (confirmado)">
    Admin reviews and confirms the order is ready for production
    
    **Actions:** Verify address, check customization notes
  </Step>
  
  <Step title="In Process (en_proceso)">
    Order is being prepared:
    - 3D printing in progress
    - Products being assembled
    - Quality checks performed
    
    **Typical Duration:** 1-3 days
  </Step>
  
  <Step title="Shipped (enviado)">
    Order shipped to customer
    
    **Actions:**
    - `fecha_envio` timestamp set
    - Tracking number added (if available)
    - Email sent with shipping notification
    
    **Delivery Time:** 3-5 days
  </Step>
  
  <Step title="Delivered (entregado)">
    Customer received the order
    
    **Actions:**
    - `fecha_entrega` timestamp set
    - Order marked complete
    - Email sent requesting review
  </Step>
</Steps>

### Timeline Tracking

Key timestamps track order progress:

- **created_at**: Order placed
- **fecha_pago**: Payment confirmed
- **fecha_envio**: Order shipped
- **fecha_entrega**: Order delivered
- **updated_at**: Last status change

<Tip>
  Calculate estimated delivery by adding 3-5 days to `fecha_envio` (shipping date).
</Tip>

## Cancelling Orders

Cancel orders before they are processed.

### Customer Cancellation

**Endpoint:** `PUT /api/client/pedidos/{id}/cancelar` (requires authentication)

<Steps>
  <Step title="Check Eligibility">
    Only orders in **"pendiente"** (pending) status can be cancelled by customers
  </Step>
  
  <Step title="Request Cancellation">
    Navigate to order details and click cancel button
    
    Or make API call to cancellation endpoint
  </Step>
  
  <Step title="Validation">
    System verifies:
    - Order belongs to your account
    - Order status is "pendiente"
    - Order hasn't been paid/processed
  </Step>
  
  <Step title="Cancellation Processing">
    If valid:
    - Order status changed to "cancelado"
    - Stock returned to products
    - `updated_at` timestamp set
    
    Success message: "Pedido cancelado correctamente"
  </Step>
</Steps>

<Warning>
  **Cannot Cancel:**
  - Orders that are **"pagado"** or beyond (payment confirmed)
  - Orders being processed, shipped, or delivered
  - Orders from other users
  
  **Error:** "Solo pedidos pendientes se pueden cancelar" (Only pending orders can be cancelled)
</Warning>

### Admin Cancellation

Administrators can cancel orders at any stage:

- Change status to "cancelado" in admin panel
- Manually refund payment if needed
- Return stock to inventory
- Notify customer via email

**Common Reasons:**
- Payment issues or fraud
- Product unavailability
- Customer request after pending window
- Order errors

## Email Notifications

Receive email updates at key order milestones.

### Notification Types

**Order Confirmation** (estado: pagado)
- Sent immediately after payment confirmation
- Includes order number, items, total, shipping address
- Confirmation that payment received

**Order Shipped** (estado: enviado)
- Sent when order dispatched
- Includes tracking number (if available)
- Estimated delivery date

**Order Delivered** (estado: entregado)
- Sent when delivery confirmed
- Thanks for purchase
- Request to leave product review

**Order Cancelled** (estado: cancelado)
- Notification of cancellation
- Reason (if provided)
- Refund information (if applicable)

**Payment Rejected** (estado: rechazado)
- Notification that payment failed
- Instructions to retry
- Link to checkout

<Note>
  All emails are sent to the `comprador_email` (buyer email) provided at checkout.
</Note>

## Shipping & Delivery

### Shipping Information

**Standard Shipping:**
- Cost: CLP 3,500 (added to all orders)
- Estimated delivery: 3-5 business days after shipment
- Delivery to address provided at checkout

**Shipping Coverage:**
- Nationwide delivery in Chile
- Remote areas may require additional time

<Note>
  Free shipping promotion available for orders over CLP 20,000 (as shown in header banner).

Once your order is shipped:

1. **Email Notification**: You'll receive shipping confirmation
2. **Tracking Number**: Included in email and order details

## Issues & Support

### Common Order Issues

**Order Not Received:**
- Check tracking information
- Verify delivery address was correct
- Check with neighbors or building reception
- Contact carrier with tracking number
- Contact Black Michi Studio support

**Wrong Item Received:**
- Take photos of incorrect item
- Keep all packaging
- Contact support with order number and photos
- Return/exchange will be arranged

**Damaged Item:**
- Document damage with photos
- Contact support within 7 days of delivery
- Include order number and product details
- Replacement or refund will be processed

**Order Status Not Updating:**
- Allow 24-48 hours for status updates
- Check spam folder for status emails
- Contact support with order number if delayed

### Contacting Support

For order assistance:

**Information to Provide:**
- Order number (Pedido #)
- Commerce Order ID (BMS-XXXXX)
- Buyer email address
- Description of issue
- Any relevant screenshots

**Support Channels:**
- Email: Contact provided in confirmation email
- Response time: Within 24-48 hours

<Tip>
  Having your order number ready speeds up the support process significantly.
</Tip>
